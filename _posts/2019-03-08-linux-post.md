# 리눅스 프로그래밍

03/08

프롬프트 `$` : 사용자의 입력을 기다림. Shell에 따라 달라짐

사용자가 입력한 명령어를 해석하는 애가 shell임.

shell : 종류가 많음. 리눅스는 besh shell을 사용함.

개발자 프롬프트 :  `#`

기본 디렉토리 : 아이디를 만들면 아이디의 홈이 생김. user1을 만들면 user1을 위한 home이 생김. 여러명 마다  home directory가 생김. 



## 명령의 구조

` 명령 [옵션] [인자] `

[옵션] 명령 다음 옵션을 줄 수 있음. 

[인자] 명령에 전달되는 인자. 옵션없이 인자만 쓸 수도 있음.



### 명령어

`ls` :  list. directory 확인하기. 

* [Option]   `-a` : all.  ` .filname` : 숨김 파일까지 볼 수 있음.

* `ls /tmp` : tmp문서에 있는 것을 조회.

`clear` : 깨끗해짐.

`date`  : 날짜를 출력해줌. 

`man` : 사용법을 보여줌. 

* e.g. man clear -> 나올 때는 q를 눌러 quit하기.

`passwd` :사용자의 비밀번호 바꿀 때.

`exit` : 터미널 종료.



### 단축키

 `ctrl + W `: 단어 지우기

`ctrl + u` : 한 문장 지우기

`ctrl + d  ` : 터미널 종료하기



## 리눅스의 파일

#### 일반파일

┌   텍스트 파일

└   바이너리 파일

텍스트 파일은 문서 편집기를 사용하여 내용을 보거나 편집할 수 있으나, 실

행 파일이나 이미지 파일의 경우 해
당 파일의 내용을 확인할 수 있는 특정 응용 프로그램이 있어야 확인할 수 있음



#### 디렉터리 (= 폴더)

리눅스는 디렉토리도 파일로 취급. 내용은 안에 있는 구조가 내용으로 들어가 있음.



#### 심벌릭 링크

윈도우의 바로가기 파일과 비슷함.



#### 장치 파일

하드디스크나 키보드 등



파일을 확인할 때에는 file을 입력

file .bashrc 를 하면 숨겨진 파일을 조회한다는 듯.

파란색은 폴더. 흰색은 일반 파일

`@` : 심벌릭 링크를 나타냄.

`/` : 디렉코리라는 것을 나타냄.

`ls /` : 루트에 있는 것을 다 보여달라

`ls -F /` : append indicator의 옵션을 사용

`~` : home 이라는 뜻. 

* `~user1` : user1 계정의 home이다.

`pwd` : print working directory. 작업 디렉토리를 출력

* 작업 디렉토리 : 내가 현재 어던 디렉토리에 있는 지에 대해.
